# release-6-detail-view-redesign — 要件定義

- リリース識別子: `release-6-detail-view-redesign`
- ステータス: 確定
- 作成日: 2026-07-03
- 対応する設計: `ARCHITECTURE.md` §10.4（詳細ビュー Agent View Lv.1）、`docs/design/class-diagram.md`（cli-ink レイヤー）

## 背景と目的

詳細ビュー（Agent View Lv.1）は project/device/branch/status/session_id/path/pr を表示しているが、罫線のないプレーンテキストの羅列で視覚的にまとまりがなく、何を表示しているビューなのか分かりにくい。また `recent_events` はスクロール手段が無く、hub 側の取得上限（`RECENT_EVENTS_LIMIT = 20`）に固定されている。

Claude Code 自身の Agent View（添付画像: レビューワークフローの進捗ペイン）を参考に、上部にプロジェクト概要のボーダー付きBOX、下部にスクロール可能なイベント履歴BOX（`X-Y of Z` の表示範囲インジケータつき）を設置し、詳細ビューを見やすく作り直す。

## スコープの確定（壁打ちでの決定事項）

| 論点                         | 決定                                                                                                                                                                                                                                                                      |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 上部BOXの表示項目            | 一覧情報（project名・device名・branch・status+経過時間）＋現状の詳細ビューが持つ session_id・path・pr・**加えて instance_id**                                                                                                                                             |
| イベント全体件数（Z）の扱い  | hub側の取得上限を **20件→100件に引き上げ**、その取得件数を「全体」として表示する（DBの真の全件数を出す count クエリは追加しない。スコープ外）                                                                                                                             |
| スクロール単位               | **j/k で1行ずつ**（一覧のカーソル移動と同じキー対応。`↑`/`↓` も同様に対応させる、後述）                                                                                                                                                                                   |
| イベント並び順               | **ターミナルログ風（古い順、最新が下端）に変更**。ただし変更はCLI表示側のみに閉じ、hub API（`GET /api/v1/instances/:id` の `recent_events`）の契約（新しい順）は変えない                                                                                                  |
| 新着イベントへの追従         | 最下部（最新）を見ている時だけ自動追従。それ以外の位置にスクロール中なら新着があっても現在位置を維持する（tail -f的動作）                                                                                                                                                 |
| 初期スクロール位置           | 詳細ビューを開いた直後は **最下部（最新イベント）** を表示                                                                                                                                                                                                                |
| 枠線スタイル                 | 上部・下部BOXとも既存の `InstanceCard`/`HelpOverlay` と同じ `borderStyle="round"` で統一                                                                                                                                                                                  |
| 表示行数                     | **ターミナル高さ（`useStdout().stdout.rows`）に応じて可変**にする（release-4 の幅対応と同様の考え方を高さにも適用）                                                                                                                                                       |
| 隣接プロジェクトへの移動     | 詳細ビューから **`←`/`→`** で一覧の並び順における前後の instance へ移動できるようにする（上下キーは本リリースでイベントスクロールに割当て済みのため別キー）。端では**反対側へ循環（wrap）**。`esc` で一覧に戻った際のカーソルは**移動先のプロジェクトに合わせて更新**する |
| BOX幅（追記）                | 概要BOX・イベント履歴BOXとも**端末幅一杯（フル幅）**にする（カードグリッドと同じ `stdout.columns` 連動の発想）                                                                                                                                                            |
| タイトル埋め込み位置（追記） | 両BOXの上辺にタイトルを埋め込む。位置は**左寄せ（角の近く）**（例: `╭─ 概要 ────────╮`）                                                                                                                                                                                  |

実装（FR-01・FR-02）後の実機PTY検証で、上部タイトル・概要BOXが「固定表示」でないと分かりにくい、BOXに何を表示しているかのタイトルが無い、というフィードバックがあり、FR-05〜FR-07として追記した（release-6 実装済み分への追加スコープ、コミット前のため同一リリースへ統合）。

## 機能要件

### FR-01: 上部プロジェクト概要BOX（優先度: 必須）

- AC-1: instance 1件の情報を `borderStyle="round"` のボーダー付きBOXにまとめて表示する。表示項目: `instance_id`・project名・device名・branch・status（色付きグリフ＋ラベル）+経過時間・session_id・path・pr状態
- AC-2: 情報源は自動更新済みの detail（取得できていれば）、無ければ一覧から渡された row をフォールバックに使う（現状の `source = detail ?? row` パターンを踏襲）
- AC-3: 状態色・グリフ・ラベル・age整形は `status-display.ts` を再利用し、表現ロジックを重複させない（§0.5 のロジック非重複方針を維持）

### FR-02: 下部イベント履歴BOX（スクロール対応）（優先度: 必須）

- AC-1: 直近イベントを `borderStyle="round"` のボーダー付きBOXに表示する。並び順はCLI表示側で反転させ、ターミナルログ風（古い順、最新が末尾）にする。hub API の `recent_events` 契約（新しい順）自体は変更しない
- AC-2: hub の直近イベント取得上限 `RECENT_EVENTS_LIMIT` を `20` → `100` に引き上げる（`src/hub/instance-status-service.ts`）。BOX内には一度に表示しきれない分をスクロールで見せる
- AC-3: 詳細ビュー表示中、`j`/`k`・`↑`/`↓`（一覧のカーソル移動と同じキー対応）でイベント履歴を1行ずつスクロールする。release-4 FR-04 AC-3 で「詳細表示中は `j`/`k`/`↑`/`↓` の入力を無視する」としていた仕様を本リリースで変更し、これらのキーはイベントスクロールに再割当てする（`1`-`5`・`Enter` は引き続き無視。誤操作でフィルタや別詳細を開かないため）
- AC-4: BOX内（右下）に「表示中の範囲（例: 1-10）of 全体件数（例: 34）」を表示する（添付画像の `1-34 of 46` と同様の書式）。全体件数は取得済みイベント件数（上限100）であり、DBの真の全件数ではない旨を実装コメントに明記する
- AC-5: 表示行数は `useStdout().stdout.rows` から算出し可変にする。`rows` が未取得（非TTY等）の場合は固定行数（例: 10行）へフォールバックする。release-4 の FR-02 AC-4（非TTYで `stdout.columns` が `undefined` になり1列フォールバックが壊れていた不具合）と同種の見落としを繰り返さないよう、非TTY相当の環境を実際に再現するテストで検証する
- AC-6: 詳細ビューを開いた直後の初期スクロール位置は最下部（最新イベント）を表示する
- AC-7: 自動更新（FR-05、既存）で新着イベントが増えたとき、現在の表示位置が最下部（最新を見ている状態）ならそのまま追従して最新を表示し続ける。それ以外の位置にスクロール中なら、新着があっても現在のスクロール位置を維持する（勝手にジャンプしない）

### FR-03: フッターのキーヒント更新（優先度: 必須）

- AC-1: 詳細ビュー中のフッターヒントに `j/k scroll`・`←/→ project`・`w wrap`（FR-08）を追加する（現状 `esc back ? help q quit` → `j/k scroll ←/→ project w wrap esc back ? help q quit` 等）

### FR-04: 詳細ビューから隣接プロジェクトへの移動（優先度: 必須）

- AC-1: 詳細ビュー表示中に `←`/`→` で、一覧の並び順（`store.filtered()`、フィルタ適用後）における前後の instance へ移動し、その詳細を表示する
- AC-2: 端（最初/最後）でさらに同方向へ進もうとした場合は反対側の端へ循環する（wrap）
- AC-3: 移動は `selectedIndex` の更新によって行う。現状の実装は `detailRow` を Enter 押下時に独立したスナップショットとして保持しているが、本リリースでは `selectedIndex` を一覧・詳細の両方が参照する単一の状態にし、表示中の instance は毎回 `store.filtered()[selectedIndex]` から導出する形に変更する。これにより `esc` で一覧に戻った際、カーソルは移動先のプロジェクトに自然に合う
- AC-4: 隣接プロジェクトへ移動した際、イベント履歴のスクロール位置（FR-02）は新しい instance の初期位置（最下部/最新、FR-02 AC-6）にリセットされる

### FR-05: 上部タイトル・概要BOXの固定表示（優先度: 必須）

- AC-1: ヘッダー（タイトル行）と概要BOX（FR-01）は、イベント履歴のスクロール操作（FR-02）によって画面上から消えたり位置がずれたりしない
- AC-2: 固定表示を構造的に保証するため、概要BOX内の各フィールド値（`instance_id`・project名・device名・branch・session_id・path・pr状態）に `wrap="truncate-end"` を付与し、長い値でも複数行に折り返してBOXの高さが変動しないようにする（`instance-card.tsx` で既に使われているのと同じパターン）。これにより `DETAIL_RESERVED_ROWS`（表示行数算出の余白計算, `event-scroll.ts`）の見積もりに対して実際のレイアウト高さが常に一致し、端末高さを超えて描画がずれる（＝ヘッダーがスクロールして見えなくなる）ことがなくなる
- AC-3: 概要BOXは常に画面上部に固定表示され、可変なのはイベント履歴BOX内の表示ウィンドウ（FR-02 AC-5）のみである
- AC-4: ~~例外として、イベント行の折り返しモード（FR-08）がONのときはこの限りではない。1件のイベントが複数行にまたがりうるため画面全体の高さが端末行数を超えることを許容し、その間はヘッダー・概要BOXがスクロールしうる（ユーザーが折り返し表示を明示的に選んだ結果として受け入れる、壁打ちでの決定）~~ → **撤回（本追記時点で再決定）**。折り返しモードでもFR-10（表示件数の動的な絞り込み）により画面全体の高さが端末行数を超えないことを保証し、例外なくヘッダー・概要BOXは常に固定表示される

### FR-06: 両BOXのタイトルを枠線に埋め込む（優先度: 必須）

- AC-1: 概要BOXの上辺に「概要」、イベント履歴BOXの上辺に「イベント履歴」のタイトルを埋め込む（例: `╭─ 概要 ──────────────────────╮`）。Ink の `Box` に枠線タイトルを埋め込むネイティブ機能は無いため、`borderTop={false}` で上辺の自動描画を無効化し、`─`/角文字を用いた自前のテキスト行をBOXの直上に描画して代替する
- AC-2: タイトル位置は左寄せ（角の近く）。左右の罫線・角文字とタイトル文字列との間には半角スペースを1つ入れる（`─ タイトル ─`）
- AC-3: BOX幅は端末幅一杯（`useStdout().stdout.columns` 連動、FR-02 の幅対応と同じ発想）にする。タイトル埋め込みの罫線文字数はBOX幅から算出する（`card-grid.ts`/`event-scroll.ts` と同様、React非依存の純粋関数として実装し単体テストで境界値を検証する）
- AC-4: イベント履歴BOXの「recent events」という現状のプレーンテキスト見出しは、このタイトル埋め込みに統合するため削除する（重複表示をしない）

### FR-07: イベント履歴のページング表示を下辺の枠線に埋め込む（優先度: 必須）

- AC-1: イベント履歴BOXの「X-Y of Z」表示範囲インジケータ（FR-02 AC-4）を、BOX内の独立した行としてではなく、下辺の罫線に埋め込む形に変更する（例: `╰──────────────────── 1-10 of 34 ─╯`）
- AC-2: 位置は右寄せ（角の近く）。タイトル（FR-06、上辺左寄せ）と対になる配置にする
- AC-3: 現状BOX内にあった `[esc] back` のヒント表示は、フッター（release-4 FR-04・release-6 FR-03 で既に `esc back` を表示済み）と重複するため削除する
- AC-4: FR-07 AC-1 の行削除により、イベント履歴BOXの表示行数の余白（`DETAIL_RESERVED_ROWS`）が変わる（従来の「見出し兼ページング行」1行分が不要になり、その分イベント表示行数が増える）。`event-scroll.ts` の余白内訳（`DETAIL_RESERVED_BREAKDOWN`）を実装に合わせて更新する

### FR-08: イベント行の折り返し/切り詰め表示切替（優先度: 必須）

イベント履歴の行（`occurred_at`・`event_type`・`tool_name`・`tool_summary` 等）が `wrap="truncate-end"` で見切れてしまい、全文を確認する手段が無いという指摘への対応。

- AC-1: イベント履歴BOX内の各イベント行は既定で切り詰め表示（`wrap="truncate-end"`、現状維持）。`w` キーでイベント行の表示モードを「切り詰め」⇔「折り返し（`wrap="wrap"`、全文表示）」にトグルする（`w` は release-4 で watch トグル撤去後は未使用のキーのため、他操作と衝突しない）
- AC-2: ~~折り返しモード（`wrap="wrap"`）のときは、1件のイベントが複数行にまたがりうるため、イベント履歴BOXの高さは可変になる（FR-05 AC-4 の例外を適用し、固定表示の保証はこの間適用しない）~~ → **撤回**。折り返しモードでもイベント履歴BOXの高さは `visible`（表示可能行数）を超えない。表示件数を動的に絞ることで固定表示を保つ（FR-10）
- AC-3: 折り返しモード中も「表示中の範囲 of 全体件数」（FR-02 AC-4・FR-07）は行数ではなく**取得済みイベント件数**を基準に算出する（1件=1行という前提が折り返しで崩れても、`X`/`Y`/`Z` の意味は変えない）
- AC-4: 詳細ビューを開いた直後の初期モードは切り詰め（現状維持）
- AC-5: モード切替状態を隣接プロジェクト移動（FR-04）や詳細ビューの再入場をまたいで保持するかどうかは実装時の判断とする（未解決事項）

### FR-09: ターミナルタイトルバーへのプロジェクト名表示（優先度: 必須）

詳細ビューでプロジェクト名を常に確認できるようにしたいという要望への対応。Ink の描画領域はターミナルの通常スクロールバック内にあり「常に画面上部に固定するペイン」を持てないため、Ink の描画とは別の場所（ターミナルのタブ/ウィンドウタイトル）に表示することで、画面内のスクロール状態によらず常に見える手段を確保する。

- AC-1: 詳細ビュー表示中、ターミナルのタブ/ウィンドウタイトルへ `project名 @ device名`（サニタイズ済み、`sanitize-display-text.ts` を再利用）を OSC エスケープシーケンスで設定する
- AC-2: 一覧ビューに戻ったら、タイトルを既定値（`Monomi`）に戻す
- AC-3: 隣接プロジェクトへの移動（FR-04）やポーリング更新で表示中の project/device 名が変わった場合、タイトルも追従して更新する
- AC-4: AppView マウント時（一覧表示中含む）にも既定値 `Monomi` をタイトルに設定する（起動前の任意のタイトルを引き継がない）
- AC-5: 非TTY環境（パイプ実行等）ではタイトル設定シーケンスの書き込み自体は無害（多くの端末エミュレータ以外では単に無視される）だが、実装は `stdout.isTTY` を問わず同じ経路で構わない（副作用は書き込みのみで害はないため、非TTY分岐は不要）

### FR-10: 折り返しモードでも画面があふれないよう表示件数を動的に絞る（優先度: 必須）

FR-08 で導入した折り返しモードは、1件のイベントが複数行にまたがることで画面全体の高さが端末行数を超え、ヘッダー・概要BOX（FR-05）がスクロールして見えなくなる問題があった。イベント件数を絞ってでも画面があふれないようにする。

- AC-1: 折り返しモード表示中、イベント履歴BOXに表示する各イベントの折り返し後行数の合計が `visible`（表示可能行数、`event-scroll.ts` の `visibleRowsForHeight`）を超えないよう、表示するイベント件数を動的に絞る
- AC-2: 絞り込みは現在の表示位置（tail-follow 中は末尾、スクロール中は `offset` 位置）を起点に、収まる件数だけを選ぶ。全件を表示しきれない場合、「X-Y of Z」の範囲ラベル（FR-02 AC-4・FR-07）は実際に表示している件数の範囲を反映する（Z は取得済み総件数のまま変えない、FR-08 AC-3 の方針を維持）
- AC-3: 折り返し後の行数見積もりは、BOXの実効幅（罫線・paddingを除いた幅）で単語境界（空白）を尊重した貪欲折り返しをシミュレーションする純粋関数で行う。外部依存は追加しない（`box-border.ts` の `displayWidth` を再利用し、`event-scroll.ts` 等へ追加する）
- AC-4: 見積もりが実際の Ink 描画（内部で `wrap-ansi` 相当のロジックを使用）より少なめに出て画面があふれることがないよう、安全側（行数を多めに見積もる）に倒す
- AC-5: 切り詰めモード（既定）の挙動・件数計算（FR-02）は変更しない。本FRは折り返しモード時のみに適用する
- AC-6: 本FRの実装により、FR-05 AC-4・FR-08 AC-2 で一時的に許容していた「折り返しモード中はヘッダー・概要BOXがスクロールしうる」という例外は撤回され、折り返しモードでも常にヘッダー・概要BOXが固定表示される

## 非機能要件

- 依存追加なし（Ink標準機能のみで実現する）
- `RECENT_EVENTS_LIMIT` の変更が既存テスト（`instance-status-service.test.ts`・`http-server.test.ts`）に影響しないことを確認する（既存フィクスチャは2〜3件程度でいずれの上限でも通るはず）
- 非TTY・`rows`未取得環境でのフォールバックを、release-4と同様に実 stdout を模したテストで検証する（`ink-testing-library` の固定値モックに頼らない）

## スコープ外（release-6 では実装しない）

- 半画面/全画面単位のページングスクロール（1行ずつのみ）
- DBの真の全イベント件数を返す count クエリの追加（取得上限の引き上げで代替）
- 上部BOXの表示項目のユーザーカスタマイズ（config等での増減）
- `RECENT_EVENTS_LIMIT` の `config.yml` での上書き（既定100固定。将来要望があれば別リリースで検討）
- 一覧画面（カードグリッド）自体の変更（対象は詳細ビューのみ）

## 未解決事項（実装中に判断）

- `useStdout().stdout.rows` から表示行数を算出する際の余白（ヘッダー・上部BOX・フッターの高さ分の差し引き）の具体的な計算式は実装時に決める
- スクロール位置の内部状態管理（イベント配列内のインデックスか、表示ウィンドウの先頭オフセットか）は実装時の設計判断とする
